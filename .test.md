# 🛡️ JANGIJOIM V1.2 실전 테스트 가이드

JANGIJOIM V1.2는 **적응형 하이브리드 DAST(Adaptive Hybrid DAST)** 아키텍처를 탑재하여, 단순 스캔을 넘어 실전적인 취약점 탐지 성능을 극대화했습니다.

---

## 🏛️ V1.2 하이브리드 아키텍처 구성

1.  **공격 표면 확장 (Katana):** 타겟 URL의 숨겨진 엔드포인트와 API를 자동으로 크롤링합니다.
2.  **고속 정밀 탐지 (Nuclei):** 크롤링된 모든 URL을 대상으로 수천 개의 보안 템플릿을 빠르게 대입합니다.
3.  **심층 정밀 분석 (OWASP ZAP):** DAST 탐지율이 SAST 결과에 비해 현저히 낮을 경우, 자동으로 ZAP 데몬을 호출하여 AJAX Spider링 및 Active Scan을 수행합니다.

---

## 🚀 실전 테스트 실행 순서

### 1. 테스트베드 및 엔진 가동
V1.2부터는 **OWASP ZAP 데몬**이 함께 실행됩니다. `docker-compose.yml`이 최신인지 확인하고 다음 명령어를 실행하십시오.

```bash
# 환경 빌드 및 백그라운드 가동
docker-compose up --build -d
```

### 2. 하이브리드 진단 파이프라인 시작
엔진 컨테이너(`gemini_engine`) 내부로 명령을 전달하여 진단을 시작합니다. 
> 💡 V1.2는 내부적으로 `Katana -> Nuclei (-> ZAP Fallback)` 순서로 작동합니다.

```bash
docker exec -it gemini_engine python main.py scan start http://juice-shop:3000 ./juice-shop-src
```

### 3. 진행 상황 실시간 모니터링
하이브리드 스캔(특히 ZAP 가동 시)은 시간이 다소 소요됩니다. 엔진 로그를 통해 현재 어떤 Tier가 작동 중인지 확인하십시오.

```bash
# 실시간 로그 확인
docker logs -f gemini_engine
```

### 4. 최종 결과 확인
모든 분석이 완료되면 `reports/` 디렉토리에 통합 보고서가 생성됩니다. 

```bash
# 생성된 보고서 목록 확인
docker exec gemini_engine ls -lh reports/
```

## 🛠️ 주요 환경변수 (선택 사항)
*   **`GEMINI_MODEL`**: 사용할 LLM 모델 (기본값: `gemini-2.5-pro`)
*   **`ZAP_URL`**: ZAP 데몬 주소 (도커 네트워크 내 기본값: `http://zap:8080`)
