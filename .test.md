# 🛡️ Jangijoim (LLM 기반 웹 취약점 진단 및 자동 패치 플랫폼)

Jangijoim은 대규모 언어 모델(LLM)을 활용하여 타겟 웹 애플리케이션의 보안 취약점을 스캔하고, 분석 및 자동 패치 가이드를 제공하는 모의해킹 자동화 CLI 도구입니다.

---

## 📋 사전 요구 사항 (Prerequisites)

로컬 개발 및 테스트를 위해 시스템에 다음 소프트웨어가 설치되어 있어야 합니다.
* **Docker** (버전 20.10 이상)
* **Docker Compose** (필수)

---

## 🚀 로컬 테스트 실행 가이드 (Docker Compose 환경)

이 프로젝트는 도커 컴포즈(Docker Compose)를 이용해 시스템 독립적이고 격리된 환경에서 모의 공격 대상(Target App)과 메인 스캐너(Engine)를 구동하여 테스트를 진행합니다. 아래 단계를 순서대로 실행하십시오.

### 1. 테스트 환경 빌드 및 가동 (백그라운드)
프로젝트 루트 디렉토리(`docker-compose.yml`이 위치한 곳)에서 다음 명령어를 실행하여 엔진과 타겟 서버 컨테이너를 생성하고 백그라운드로 실행합니다.

```bash
docker-compose up -d --build
```

### 2. 취약점 스캔(공격) 파이프라인 시작
두 서버가 모두 구동된 상태에서, 엔진 컨테이너(`gemini_engine`) 내부로 CLI 명령을 전달하여 타겟 컨테이너(`target-app`)에 대한 공격 테스트를 시작합니다.
> 💡 도커 네트워크(`sec-network`)로 통신하므로, 타겟 URL은 `127.0.0.1`이 아닌 `http://target-app:3000`을 사용해야 합니다.

```bash
docker exec -it gemini_engine python main.py scan start http://target-app:3000 ./target_app_mock
```

### 3. 테스트 종료 및 환경 정리
모든 테스트가 끝난 후 환경을 깔끔하게 정리하고 싶을 때는 아래 명령어를 사용하여 구동 중인 컨테이너와 네트워크를 삭제합니다.

```bash
docker-compose down
```

## 🛠️ 주요 명령어 (CLI Usage)

엔진 컨테이너를 통해 사용할 수 있는 `main.py`의 기본 CLI 명령어입니다.

* **도움말 확인:**
  ```bash
  docker exec -it gemini_engine python main.py --help
  ```
* **스캔 모듈 단독 실행 정보 확인:**
  ```bash
  docker exec -it gemini_engine python main.py scan --help
  ```
